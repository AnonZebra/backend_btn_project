When you've clicked the button, take a look in the terminal/bash/console window that you ran python manage.py runserver in. You should see a message.

To understand the chain that happens:

  1. Look in 'backend_btn_project/urls.py'. There you see that the include function is used to reroute requests to 'buttons/urls.py'.
  2. Look in 'buttons/urls.py'. There you see that the view buttons_home_view is imported from 'views.py' and used.
  3. Look in 'buttons/views.py'. The function buttons_home_view checks if the request has the method "POST" (i. e. data from a form with attribute method='post' has been submitted with the request). If this is true, the function say_hi_in_terminal (imported from 'buttons/button_functions.py' is called. In either case, buttons_home_view returns a rendered page based on the html template located at 'buttons/templates/buttons/buttons_home.html'.
  4. Look in the html template 'buttons/templates/buttons/buttons_home.html' (or right click on the button here and select 'inspect element'). You'll see that a very simple form is used, like this: 
    <form method="post">
{% csrf_token %}
<input type="submit" value="Click me!">
</form>

    So no particular data is sent by the form - it just makes sure that a request using the "POST" method is sent.

  5. Look inside 'buttons/button_functions.py'. There you see what the function say_hi_in_terminal does (it simply prints a message).
  6. And that's it!

Of course, printing a message to the console isn't very impressive. But you could easily replace the say_hi_in_terminal function with whatever function you like of course (just remember to import it). At the same time, the particular method used here (making whatever "POST" method request trigger the function being called) probably is very bad practice.
